# 1. TypeScript Overview

타입스크립트의 탄생 배경과 주요 특징 및 장점

## 웹 개발의 역사

### Javasciprt의 탄생

- 1995년, 넷스케이프 커뮤니케이션즈의 브렌던 아이크(Brendan Eich)에 의해 탄생
- 브렌던 아이크는 웹의 다양한 콘텐츠를 표현하기 위해 이미지, 플러그인 요소를 추가할 수 있는 스크립트 언어를 개발하고자 함
- C, Java와 유사한 문법을 가지고 있으며, 별로 유명하지 안던 객체 지향 언어인 Self 언어의 프로토타입 기반 상속 개념과 Lisp 언어 중 하나인 Scheme 언어의 일급 함수 개념을 차용한 경량의 프로그래밍 언어
- 인터넷 익스플로어는 Javasciprt에 대응하기 위해 JScript라는 이름으로 자체 구현

### Javascript의 표준, ECMAScript의 탄생

- 처음 각 브라우저마다 DOM 구조가 완전히 달라, 브라우저마다 웹 페이지가 다르게 동작하거나 제대로 동작하지 않는 크로스 브라우징 이슈가 발생
- 따라서 개발자는 각 브라우저에 맞게 코드를 작성해야 했으며, 이는 개발 생산성을 떨어뜨리는 요인이 되었음
- 초기 자바스립트는 브라우저 생태계를 고려해서 작성된 것이 아니였고 단지 새로운 기능이 추가되는 형태로 발전
  - 이에 따라 자바스크립트와 브라우저의 발전 속도 간 차이가 발생
- 자바스크립트에 어떤 기능이 추가된다면 _runtime_ 환경인 브라우저도 이 기능을 지원할 수 있어야 함
  - 이 문제를 해결하기 위해 자바스크립트에 **polyfill**과 **transpile**가 같은 개념이 등장
- 따라서 크로스 브라우징 ㅇ이슈를 해결하기 위해 자바스크립트의 표준을 정의하고자 하는 노력이 시작되었으며, 이를 위해 **_ECMA International(국제 표준화 기구)_**이라는 기구에서 **ECMAScript**라는 표준을 정의

---

💡 **폴리필(polyfill)과 트랜스파일(transpile)**

- **폴리필(polyfill)**: 브라우저가 지원하지 않는 코드를 브라우저에서 사용할 수 있도록 변환한 코드 조각이나 플러그인
- **트랜스파일(transpile)**: 최신 버전의 코드를 예전 버전의 코드로 변환하는 과정
- 둘 다 최신 기능을 구버전의 실행 환경에서 동작할 수 있게 바꿔주는 역할
- 대표 폴리필 라이브러리로는 `core-js`, `babel-polyfill` 등이 있으며, 대표 트랜스파일러로는 `Babel`이 있음

### 웹 사이트 vs 웹 애플리케이션

#### 웹 사이트

- 웹사이트는 수집된 데이터 및 정보를 특정 페이지에 표시하기 위한 목적으로 만들어진 **정적인 웹 페이지**
- 단뱡향으로 정보를 제공하며, 사용자와의 상호 작용하지 않음
- HTML에 링크가 연결된 웹 페이지 모음으로 콘텐츠가 동적으로 업데이트 되지 않음

#### 웹 애플리케이션

- 웹 애플리케이션은 사용자와 **상호 작용하는 양방향 소통의 웹**
- 검색, 로그인, 데이터 입력, 데이터 검색, 데이터 수정, 데이터 삭제 등의 기능을 제공

### 개발 생태계의 발전

- 거대한 웹 애플리케이션을 등장함에 따라 컴포넌트 단위 개발 방식의 도입
- CBD는 서비스에서 다루는 데이터를 구분하고 그에 맞는 UI를 표현할 수 있게 컴포넌트 단위로 개발하는 접근 방식
  - 작은 컴포넌트를 조합해서 큰 컴포넌트를 만들어내는 방식

#### 컴포넌트 vs 모듈

- 컴포넌트는 모듈과 유사하게 하나의 독립된 기능을 재사용하기 위한 코드 묶음
  - But, 모듈과 달리 런타임 환경에서 독립적으로 배포, 실행될 수 있는 단위
  - 컴포넌트는 다른 컴포넌트와의 의존성을 최소화하거나 없애야 함

**💡 컴포넌트 베이스 개발(Component Based Development, CBD)**
재사용 가능한 컴포넌트를 만들어서 조립하는 방식으로 개발하는 방법론

## Javascript의 한계

### 동적 타입 언어

- 자바스크립트는 동적 타입 언어로, 변수의 타입을 코드가 실행되는 런타임 시점에 결정

### 동적 타이핑 시스템의 한계

```javascript title="정상 동작" showLineNumbers
// a, b에 숫자 합을 반환 하는 함수
const sumNumber = (a, b) => a + b;

sumNumber(1, 2); // 3
```

```javascript title="비정상 동작" showLineNumbers
// a, b에 숫자 합을 반환 하는 함수
const sumNumber = (a, b) => a + b;

sumNumber(100); // NaN
sumNumber("1", "2"); // '12'
```

- 자바스크립트는 동적 타입 언어이기 때문에 함수의 매개변수에 대한 타입을 명시적으로 지정하지 않음
- 동적 타입 언어라는 특성 때문에 sumNumber 함수를 호출할 때 사용되는 매개변수의 타입을 런타임 시점에 결정
- 이로 인해 개발자 의도와 다르게 함수의 매개변수에 대한 타입을 잘못 지정하거나, 함수의 반환값이 예상과 다르게 나올 수 있음

### 타입스크립트의 등장

- 타입스크립트는 자바스크립트의 superset 언어, 자바스크립트 코드를 그대로 사용 가능

**💡 슈퍼셋(superset)**

- 기존 언어에 새로운 기능과 문법을 추가해 보완하거나 향상시킨 언어
- 기존 언어와 호환되며 일반적으로 컴파일러 등으로 기존 언어 코드로 반환되어 실행

---

#### Typescript 특징

- **안전성 보장**: 정적 타입 언어로, 컴파일 시점에 타입을 검사하여 런타임 오류를 방지
- **개발 생산성 향상**: IDE에서 타입 자동 완성 기능을 제공하여 변수와 함수 타입을 추론 가능
- **코드 가독성 & 유지보수성 향상**: 타입을 명시적으로 선언하여 코드 의미를 명확하게 표현하며, 코드의 재사용성과 유지보수성을 높임
- **협업에 유리**: 타입을 명시적으로 선언하여 코드의 의도를 명확하게 전달하기 때문에 협업을 원활하게 함
- **자바스크립트에서 점진적으로 적용 가능**: 타입스크립트는 자바스크립트의 슈퍼셋이기 때문에 일괄 전환이 아닌 점진적 도입이 가능

## 결론

- 타입스크립트는 자바스크립트의 한계를 극복하기 위해 등장한 **정적 타입** 언어
- 타입스크립트는 개발 생산성 향상과 코드 안정성을 보장하는 장점을 가지고 있음
- 타입스크립트는 자바스크립트의 **슈퍼셋**이기 때문에 기존 자바스크립트 코드를 그대로 사용 가능하며, 점진적으로 도입이 가능
- 따라서 타입스크립트는 대규모 웹 애플리케이션 개발에 적합한 언어로 평가받고 있음
